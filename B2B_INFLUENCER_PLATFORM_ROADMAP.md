# B2B 인플루언서 관리 시스템 로드맵 메모

## 목표 정의

현재 목표는 단순 인플루언서 검색 도구가 아니라, 병원/에이전시가 인플루언서를 발굴하고 관리하고 협업 성과를 추적하는 B2B SaaS를 만드는 것이다.

지향하는 최종 제품 형태는 아래와 같다.

1. 병원/대행사가 조건에 맞는 인플루언서를 검색한다.
2. 검색된 인플루언서를 리스트, 태그, 상태값으로 관리한다.
3. 병원 계정 또는 대행사 계정으로 아웃리치를 보낸다.
4. 협찬/캠페인 기간 동안 게시 여부와 성과를 추적한다.
5. 계약, 응답률, 진행 상태, 성과를 누적 데이터로 관리한다.

즉, 제품의 본질은 "인플루언서 검색 엔진"이 아니라 "인플루언서 CRM + Outreach + Campaign Tracking"이다.

---

## 1. 인플루언서 시딩 DB 수집 전략

### 왜 필요한가

실시간 검색만으로는 제품성이 약하다. 병원이 원하는 것은 검색 속도와 결과 품질, 그리고 반복적으로 사용할 수 있는 누적 데이터다. 따라서 시딩 DB를 운영해야 한다.

### 권장 방향

수집 전략은 한 번에 크게 가져오는 방식보다, 계층형 파이프라인으로 운영하는 것이 낫다.

1. Seed Pool 구축
   이미 알려진 인플루언서 후보를 대량으로 확보한다.
   소스는 해시태그 기반 수집, 경쟁 병원 언급 계정, 기존 수동 등록 계정, 공개 프로필 기반 확장 탐색이다.

2. Enrichment
   후보 계정의 프로필, 팔로워 수, 카테고리, 최근 게시물, 평균 반응, 최근 활동일을 정규화해 저장한다.

3. Scoring
   병원 관점에서 의미 있는 점수를 만든다.
   예시는 적합도, 활동성, 협찬 적합도, 병원/뷰티 카테고리 일치도, 허수 가능성, 최근 업로드 빈도다.

4. Refresh
   전 계정을 같은 주기로 다시 긁지 말고 티어별 주기를 둔다.

### 추천 수집 주기

1. Hot Tier
   최근 검색에 자주 등장하거나 캠페인 진행 중인 계정.
   1일 1회 또는 3일 1회 갱신.

2. Warm Tier
   최근 30일 내 검색되었지만 활성 캠페인은 없는 계정.
   1주 1회 갱신.

3. Cold Tier
   장기 보관 후보군.
   1개월 1회 또는 필요 시 갱신.

### 추천 데이터 모델

핵심 엔터티는 아래 수준까지는 분리하는 것이 좋다.

1. influencers
   핸들, 플랫폼, 카테고리, 팔로워, 소개, 상태, 최근 수집일

2. influencer_metrics_snapshots
   날짜별 팔로워, 평균 좋아요, 평균 댓글, ER, 최근 업로드일

3. influencer_posts
   게시물 URL, 게시일, 캡션, 해시태그, 좋아요/댓글/조회수

4. influencer_tags
   성형, 피부, 치과, 지역, 마이크로/미드/매크로 등 내부 분류

5. influencer_match_scores
   병원 업종별/캠페인별 적합도 계산 결과

### 내 의견

이 부분은 반드시 먼저 해야 한다. 이유는 DM 기능이나 캠페인 추적은 결국 좋은 후보 DB가 있어야 의미가 생기기 때문이다. 지금 프로젝트 기준으로는 검색 API는 있지만, "누적 운영 데이터" 레이어가 약하다. 따라서 1순위는 검색 결과를 실시간 응답으로 끝내지 말고 시딩 DB에 축적하는 구조로 바꾸는 것이다.

## 2. 병원/업체 인스타그램 계정 등록 후 DM 발송

### 제품적으로 필요한 이유

병원이 원하는 것은 "찾기"보다 "연락하기"까지 한 번에 되는 흐름이다. 따라서 계정 연결과 메시지 발송은 제품 가치가 매우 크다.

### 전제 정리

여기서 말하는 DM은 대량 자동 발송이 아니라, 병원 또는 대행사 마케팅팀이 이 시스템 안에서 인플루언서를 선택하고 직접 DM을 보내는 운영형 기능이다.

즉, 제품 요구사항은 아래에 가깝다.

1. 병원 계정을 시스템에 연결한다.
2. 검색된 인플루언서를 선택한다.
3. 우리 서비스의 UI에서 메시지를 작성하거나 템플릿을 불러온다.
4. 마케팅팀 담당자가 직접 발송한다.
5. 발송 이력과 응답 상태를 시스템에 남긴다.

### 현실적인 이슈

여기서 가장 중요한 것은 기술보다 정책과 권한이다.

자동화가 아니어도, 우리 서비스 UI에서 DM을 보낸다는 것은 결국 Instagram 메시징 권한과 계정 연결 구조가 필요하다는 뜻이다. 따라서 공식 Instagram API로 가능한 범위와 불가능한 범위를 먼저 분리해야 한다.

즉, 아래 두 가지를 구분해야 한다.

1. 제품 요구사항
   "병원 계정으로 인플루언서에게 직접 메시지를 보내고 싶다"

2. 실제 구현 가능 방식
   공식 API 범위 안에서 가능한가
   아니면 브라우저 자동화/비공식 방식이 필요한가

### 권장 접근

가장 안전한 방향은 "DM 자동화 시스템"이 아니라, "DM 발송이 포함된 아웃리치 워크플로우 시스템"으로 설계하는 것이다.

#### 권장 1단계

1. 병원 계정 등록
   인스타 계정 식별 정보, 브랜드명, 담당자, 템플릿 메시지 저장

2. 타겟 리스트 생성
   검색 결과에서 캠페인 대상으로 저장

3. 메시지 작성/검수 UI
   발송 대상, 템플릿, 담당자, 상태값 관리

4. 발송 이력 저장
   누가 언제 어떤 계정으로 어떤 인플루언서에게 보냈는지 기록

이 단계만으로도 CRM 제품 가치는 충분히 생긴다.

#### 권장 2단계

Meta/Instagram 공식 기능 범위 내에서 실제 발송 가능한 메시징 연동이 있는지 검토한다. 가능하다면 공식 연동을 우선한다.

#### 비권장 단기 접근

비공식 로그인 자동화, 세션 의존형 브라우저 조작, 우회 발송 구조는 계정 정지와 서비스 리스크가 크다.

### 내 의견

이 기능은 중요하지만, 구현의 중심은 여전히 메시지 자동화가 아니라 계정 연결과 발송 워크플로우다. 초기에 무리해서 우회성 DM 전송에 기대면 전체 서비스가 인스타 정책 의존형이 된다. 그래서 초기 버전은 아래처럼 가는 것이 맞다.

1. 병원 계정 등록
2. 아웃리치 대상 관리
3. DM 템플릿 관리
4. 발송 이력/상태 관리
5. 가능 범위 내 공식 연동

즉, "자동화"보다 먼저 "실사용 가능한 DM 운영 시스템"을 만들 것을 권장한다.

## 3. 협찬 기간 내 게시 여부 추적

### 왜 중요한가

이 기능이 있어야 제품이 단순 CRM이 아니라 성과 관리 도구가 된다. 병원 입장에서는 누가 실제로 업로드했는지, 약속한 조건을 지켰는지, 반응이 어땠는지가 중요하다.

### 권장 기능 범위

1. 캠페인 생성
   병원, 인플루언서, 시작일, 종료일, 요구 플랫폼, 요구 게시물 타입, 필수 해시태그, 멘션 규칙 저장

2. 게시물 탐지
   협찬 기간 동안 해당 인플루언서의 신규 게시물/릴스를 주기적으로 수집

3. 매칭 로직
   아래 기준으로 캠페인 게시물 여부를 판별
   계정 일치, 기간 일치, 해시태그 일치, 멘션 일치, 링크 일치, 키워드 일치

4. 상태 관리
   미업로드, 업로드 감지, 검수 필요, 승인, 미충족

5. 성과 추적
   좋아요, 댓글, 조회수, 게시물 URL, 업로드 시점, 예상 도달

### 추천 운영 방식

캠페인 단위 스케줄러가 필요하다.

1. 캠페인 진행 중
   1일 1회 또는 하루 2회 수집

2. 업로드 감지 후 7일
   성과 변화 추적을 위해 더 자주 갱신

3. 캠페인 종료 후
   최종 성과 스냅샷 저장

### 추천 데이터 모델

1. campaigns
2. campaign_influencers
3. outreach_messages
4. tracked_posts
5. campaign_post_matches
6. performance_snapshots

### 내 의견

이 기능은 실제 돈을 내게 만드는 핵심 기능이다. DM보다도 제품 차별화 포인트가 명확하다. 검색과 CRM은 경쟁자가 많지만, "협찬 진행 상태와 업로드 여부를 병원 관점에서 자동 추적"하는 기능은 훨씬 사업 가치가 높다.

따라서 우선순위는 아래가 더 적절하다.

1. 인플루언서 시딩 DB
2. 캠페인/협찬 추적
3. 아웃리치 운영 시스템
4. DM 기능 연동 고도화

## 전체 권장 제품 로드맵

### Phase 1. Discovery DB

목표는 검색 가능한 인플루언서 DB를 만드는 것이다.

1. 검색 결과를 DB에 누적 저장
2. 프로필/게시물 스냅샷 저장
3. 카테고리/지역/업종 태깅
4. 적합도 점수 계산

### Phase 2. CRM

목표는 병원이 후보군을 실제 운영 가능한 상태로 관리하게 만드는 것이다.

1. 즐겨찾기, 리스트, 태그
2. 담당자 메모
3. 상태값 관리
   예: 후보, 연락 예정, 협의 중, 진행 중, 완료, 보류

### Phase 3. Campaign Tracking

목표는 협찬 진행과 업로드 결과를 추적하는 것이다.

1. 캠페인 생성
2. 인플루언서 연결
3. 기간 내 게시물 탐지
4. 성과 스냅샷

### Phase 4. Outreach

목표는 연락 프로세스를 시스템화하는 것이다.

1. 메시지 템플릿
2. 발송 이력 관리
3. 수동 발송 지원
4. 정책 검증 후 공식 연동 검토

### Phase 5. Automation

자동화보다 운영 품질 고도화가 마지막 단계에 가깝다.

1. 타겟 추천
2. 유사 인플루언서 추천
3. 응답 가능성 예측
4. 캠페인 성과 예측

## 최종 의견

지금 방향은 맞다. 다만 제품의 중심을 "인스타 DM 자동화"로 보면 안 되고, "마케팅팀이 실제로 쓸 수 있는 인플루언서 운영 툴"로 봐야 한다.

내 판단으로는 아래처럼 생각하는 것이 가장 현실적이다.

1. 핵심 자산은 인플루언서 데이터베이스다.
2. 핵심 매출 기능은 캠페인 추적이다.
3. 핵심 운영 기능은 CRM이다.
4. DM 기능은 공식 연동 가능 범위를 기준으로 붙여야 한다.

즉, 피처링 같은 플랫폼을 목표로 한다면 먼저 만들어야 하는 것은 "발송 자동화"가 아니라 "신뢰할 수 있는 인플루언서 운영 시스템 + 실사용 가능한 DM 워크플로우"다.
